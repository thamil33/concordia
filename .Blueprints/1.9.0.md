#  1.9.0 Release Notes pysim 0.0.1

## Major Additions

### Document System

- Provided a full guide to the document abstraction and interactive document prompting utilities.

### New API Integrations

- **OpenRouter API Implementation**
  - Added `openrouter_model.py` for direct integration with the OpenRouter API.
  - Supports both free-form text and multiple choice via HTTP requests.
  - Reads API key and endpoint from environment variables or arguments.
  - Handles timeouts, error reporting, and response parsing.

- **LMStudio API Implementation**
  - Added `lmstudio_model.py` for integration with the LMStudio local API.
  - Communicates with LMStudio via HTTP (default: `http://localhost:1234/v1`).
  - Supports both text and multiple choice via prompt formatting.
  - Reads API key and base URL from environment variables if not provided.
  - Handles timeouts and error reporting.

## Major Refactor: GameMaster → Orchestrator

- **Renamed all references of "GameMaster" to "Orchestrator"** throughout the codebase, documentation, and guides.
- Updated class names, variable names, and documentation to reflect the new terminology.
- All orchestrator logic, prefab roles, and scene specifications now use "orchestrator" as the canonical term.
- This change clarifies the role of the orchestrator as the central controller and narrative resolver in simulations.

---

**This release is focused on documentation, developer experience, new API integrations (OpenRouter and LMStudio), major terminology refactoring (GameMaster → Orchestrator), and environment management. No breaking changes to core APIs.**
